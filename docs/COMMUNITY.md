# Community Guidelines

Welcome to the ICDFA Open Labs community! This document outlines how we work together to create a welcoming, inclusive, and productive environment.

## Our Community Values

**Respect:** We value diverse perspectives and treat all community members with respect and dignity.

**Collaboration:** We believe in the power of working together to achieve common goals.

**Learning:** We foster a culture of continuous learning and knowledge sharing.

**Inclusivity:** We welcome people of all backgrounds, experience levels, and identities.

**Integrity:** We conduct ourselves with honesty and transparency.

## Code of Conduct

All community members must adhere to our [Code of Conduct](../CODE_OF_CONDUCT.md). This includes:

- Being respectful and inclusive
- Avoiding harassment and discrimination
- Accepting constructive criticism
- Focusing on what's best for the community
- Reporting violations appropriately

## How to Participate

### Join the Conversation

Participate in our community through:

- **GitHub Discussions:** Ask questions, share ideas, and discuss labs
- **GitHub Issues:** Report bugs, suggest features, and track work
- **Pull Requests:** Contribute code and documentation improvements
- **Email:** Contact us at contact@icdfa.com

### Ask Questions

We encourage questions! When asking:

1. Search existing discussions first
2. Provide clear context and details
3. Be specific about what you need help with
4. Share what you've already tried
5. Be patient and respectful

### Share Knowledge

Help others by:

- Answering questions in discussions
- Sharing your lab experiences
- Writing tutorials or guides
- Providing feedback on labs
- Mentoring newer members

### Report Issues

Found a bug or problem? Help us improve:

1. Check if it's already reported
2. Provide clear reproduction steps
3. Include your environment details
4. Attach error messages or logs
5. Be constructive and helpful

### Contribute

Ready to contribute? See [CONTRIBUTING.md](../CONTRIBUTING.md) for:

- How to submit labs
- Code standards and practices
- Review process
- Recognition and credit

## Community Channels

### GitHub Discussions

**Purpose:** General discussion, questions, and community support

**Topics:**
- Getting started questions
- Lab recommendations
- Learning paths and progression
- General cybersecurity discussion
- Community announcements

**Guidelines:**
- Search before posting
- Use clear titles
- Provide context
- Be respectful and constructive

### GitHub Issues

**Purpose:** Bug reports, feature requests, and task tracking

**Types:**
- Bug reports
- Feature requests
- Lab improvements
- Documentation updates
- General tasks

**Guidelines:**
- Use templates provided
- Include all relevant information
- Search for duplicates
- Be specific and detailed

### Pull Requests

**Purpose:** Contribute code and documentation

**Process:**
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request
5. Address feedback
6. Get merged!

**Guidelines:**
- Follow contribution guidelines
- Include clear descriptions
- Reference related issues
- Keep commits clean
- Be responsive to feedback

### Email

**Contact:** contact@icdfa.com

**Use for:**
- General inquiries
- Partnership opportunities
- Security concerns
- Sensitive matters

## Community Standards

- Avoid offensive jokes or comments
- Respect different opinions
- Listen actively
- Assume good intent

### Constructive Feedback

- Be specific and actionable
- Focus on the work, not the person

### Inclusive Participation

- Accommodate different needs
- Celebrate differences
- Foster belonging

## Conflict Resolution

### Addressing Issues

If you encounter conflict or problems:

1. Try to resolve directly with involved parties
2. Seek mediation from moderators
3. Report to conduct@icdfa.com if needed
4. Follow the code of conduct enforcement process

### Reporting Violations

Report code of conduct violations:

- Email: conduct@icdfa.com
- Include specific details
- Provide context and evidence
- Maintain confidentiality
- Allow time for investigation

## Community Moderation

### Moderation Team

Our moderation team ensures community standards:

- Enforce code of conduct
- Facilitate discussions
- Resolve conflicts
- Remove inappropriate content
- Support community members

### Moderation Actions

Actions may include:

- Warning or reminder
- Comment removal
- Temporary suspension
- Permanent ban

**Due process:** Members can appeal decisions
